Django vs Flask vs FastAPI

Django vs Flask vs FastAPI // SCHEMATIC
點節點看說明；點框架名稱跳到詳細卡片。訊號脈衝＝request 流：FastAPI 三顆並行快速 (async)，Django / Flask 單顆依序 (sync)。

Django

FULL-STACK FRAMEWORK

定位
全功能 full-stack，ORM / admin / auth 內建
架構
WSGI，3.x 起支援 ASGI
內建
ORM、admin、auth、migration 全包
效能
中等
適用
完整網站、後台、regulated 行業
有 DRF (Django REST Framework) 可做 API，但比 FastAPI 重很多，除非已在 Django 生態才考慮。

Flask

MICRO-FRAMEWORK

定位
極簡 micro-framework，什麼都自己挑
架構
WSGI、同步
內建
幾乎沒有，靠 extension
效能
~2–3k RPS
適用
小服務、MVP、教學
Flask 3.x 其實也能掛 async view，但底層仍 WSGI，效能不會變 FastAPI。

FastAPI

ASYNC-FIRST API FRAMEWORK

定位
API-first、高效能，自動生 OpenAPI docs
架構
ASGI、原生 async/await
內建
validation (Pydantic)、OpenAPI/Swagger 自動文件
效能
~15–20k RPS (Uvicorn)，高併發 / IO-bound 最強
適用
純 API、microservice、AI/LLM 整合
以 SDET 背景寫測試工具 / mock server，FastAPI 幾乎是最佳解：Pydantic schema 驗證 + 自帶 Swagger UI 方便同事調試。

選型一句話

要做完整網站（含後台管理）DJANGO
只是小工具或想完全掌控FLASK
新的純 API 專案（尤其 AI 相關）FASTAPI

FastAPI 在 2026 已是新 API 專案的預設選擇。
